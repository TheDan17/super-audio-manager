### Реализованные паттерны
1.  **Singleton (Одиночка):**
    *   *Где:* Класс `Core` (`core/core.h`).
    *   *Зачем:* Обеспечивает единую точку доступа к данным приложения (библиотека, плеер, настройки) из любой части интерфейса.
2.  **Strategy (Стратегия) / Interface:**
    *   *Где:* Интерфейс `IModule` и его реализации (`modules/impl_modules.h`).
    *   *Зачем:* Позволяет подменять алгоритмы и поведение (разные модули) без изменения кода, использующего их.
3.  **Facade (Фасад):**
    *   *Где:* Класс `Core`.
    *   *Зачем:* Предоставляет простой интерфейс (`scanDirectory`, `playTrack`) для сложной подсистемы (Qt Multimedia, QSettings, File I/O).
4.  **Observer (Наблюдатель):**
    *   *Где:* Механизм сигналов и слотов Qt (`signals:`, `connect`).
    *   *Зачем:* `Core` уведомляет UI об изменениях (`queueChanged`, `trackChanged`) без жесткой связности.
