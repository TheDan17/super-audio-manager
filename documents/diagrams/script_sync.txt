Имя Use Case: 
	Синхронизировать с устройством
Актеры: 
	- Владелец библиотеки (Инициатор) 
	- Модуль Синхронизации (Основной обработчик)
Предусловия:
	- В системе создан как минимум один ПрофильСинхронизации.
	- Папки-источники библиотеки проиндексированы.
	- Целевое устройство (если выбрано) физически подключено.
	- Фильтры существуют (если выбраны) в приложении.
Постусловия:
	- Файлы, соответствующие фильтрам профиля, скопированы/перемещены на целевое устройство, пройдя все этапы конвейера.
	- Пользователю отображена сводка об успешном завершении операции.

ОСНОВНОЙ ПОТОК СОБЫТИЙ (Действие актера - Действие системы)

Шаг 1.  Актер открывает Модуль Синхронизации - Система отображает список сохраненных Профилей Синхронизации.
Шаг 2.  Актер выбирает Профиль Синхронизации (например, "Мой Плеер") и нажимает "Начать Синхронизацию" - Система инициирует процесс Валидации Профиля.
Шаг 3.  - Система, используя фильтры профиля (F-SYNC-2-1), генерирует список исходных Треков и сравнивает их с файлами в цели (F-SYNC-4-2).
Шаг 4.  - Система формирует и отображает План Синхронизации (таблица: Новые/Обновить/Удалить-с-Цели).
Шаг 5.  Актер просматривает План и подтверждает его кнопкой "Запустить" - Система начинает выполнение. 
Шаг 6.  - Для каждого файла из плана:
Шаг 7.  - Система запускает Конвейер Обработки (F-SYNC-3-1).
Шаг 8.  - Система копирует обработанный временный файл в место назначения (F-SYNC-4-1).
Шаг 9.  - Система обновляет Статус-бар, отображая Сегментированный Прогресс-бар (NF-USE-5).
Шаг 10.  - После копирования всех файлов, система удаляет временные копии и, если опция включена (F-SYNC-4-3), удаляет "осиротевшие" файлы с цели.
Шаг 11.  - Система отображает диалоговое окно "Синхронизация завершена" и показывает Сводку (NF-USE-6).

АЛЬТЕРНАТИВНЫЙ ПОТОК СОБЫТИЙ

Шаг 2.1.  - Система не может обнаружить Целевое Устройство/Папку.
Шаг 2.2.  - Система открывает единое диалоговое окно проблем (F-SYNC-5-2). Сообщение включает ошибку "Устройство не найдено" и подсказку (F-SYNC-5-3).
Шаг 2.3.  Актер в окне проблем исправляет путь или подключает устройство и нажимает "Перепроверить". -
Шаг 2.4.  Система повторяет шаг 2. Если проблема решена - вернуться к Шагу 3 Основного потока. Если нет, вернуться к Шагу 2.2.

Шаг 4.1.  - Актер в Плане Синхронизации видит неожиданное количество удалений/обновлений.
Шаг 4.2.  Актер нажимает "Отмена" - Система завершает процесс, сохраняет профиль, но не выполняет никаких файловых операций. Завершение Use Case.

ИСКЛЮЧИТЕЛЬНЫЙ ПОТОК СОБЫТИЙ

Шаг 7.1.  Один из этапов конвейера (например, Модуль Конвертации) возвращает критическую ошибку для файла.
Шаг 7.2.  Система логирует ошибку, пропускает файл, продолжает выполнение цикла для следующих файлов (шаг 9). Ошибка будет отражена в финальной Сводке (NF-USE-7).
